jsonパッケージ

Goを触ってる人ならだれもが一度はお世話になるであろう パッケージ「json」

今回はそんな json パッケージについて、
知っておくと便利な機能を2つ紹介します。

（比較的有名なものしかないですが🙏）


1. 独自の変換ロジックを実装できる

例えば、下記のコードのように、
ある構造体（Human）のフィールドを外部公開したくない場合、
jsonパッケージの Unmarshal()，Marshal() が使えません。
（上記関数は外部公開されたフィールドのみ変換できる）

 1type Human struct {
 2	// フィールドを外部公開したくない
 3	name string
 4	age  int
 5}
 6
 7func main() {
 8	h := Human{
 9		name: "Taro",
10		age:  21,
11	}
12
13	// 構造体 → JSON
14	j, _ := json.Marshal(h)
15	fmt.Println(string(j)) // {}
16
17	// JSON → 構造体
18	var uh Human
19	_ = json.Unmarshal(j, &uh)
20	fmt.Printf("%+v\n", uh) // {name: age:0}
21}

playgroud


実際に実行してみると、うまく変換できていないことが分かると思います。

さて、このときどうすれば正しく変換できるかというと、
変換対象の構造体に以下のメソッドを生やしてやればOKです。

  • MarshalJSON() ([]byte, error)
  • UnmarshalJSON(data []byte) error
 1type Human struct {
 2	// フィールドを外部公開したくない
 3	name string
 4	age  int
 5}
 6
 7func (h Human) MarshalJSON() ([]byte, error) {
 8	type tmp struct {
 9		Name string `json:"name"`
10		Age  int    `json:"age"`
11	}
12
13	t := tmp{
14		Name: h.name,
15		Age:  h.age,
16	}
17	return json.Marshal(t)
18}
19
20func (h *Human) UnmarshalJSON(data []byte) error {
21	type tmp struct {
22		Name string `json:"name"`
23		Age  int    `json:"age"`
24	}
25
26	var t tmp
27	_ = json.Unmarshal(data, &t)
28
29	h.name = t.Name
30	h.age = t.Age
31	return nil
32}
33
34func main() {
35	h := Human{
36		name: "Taro",
37		age:  21,
38	}
39
40	// 構造体 → JSON
41	j, _ := json.Marshal(h)
42	fmt.Println(string(j)) // {"name":"Taro","age":21}
43
44	// JSON → 構造体
45	var uh Human
46	_ = json.Unmarshal(j, &uh)
47	fmt.Printf("%+v\n", uh) // {name:Taro age:21}
48}

うまく変換できましたね👍

このように、Marshal() と Unmarhsal() は
対象の構造体に生えている MarshalJSON() と UnmarhsalJSON() を見に行ってくれるわけです。

2. 構造体なしでも変換できる

実はJSONの変換処理は、構造体をきっちり定義してやる必要はありません。

以下に示すとおり、
interface{} を使うと、map[string]interface{} に変換してくれます。

 1func main() {
 2	// どんな内容か分からないJSON
 3	mysteriousJSON := "{\"name\": \"Taro\", \"age\": 21}"
 4
 5	var i interface{}
 6	_ = json.Unmarshal([]byte(mysteriousJSON), &i)
 7
 8	for k, v := range i.(map[string]interface{}) {
 9		fmt.Printf("%s: %v\n", k, v) // キー・バリューのセットを取得できる
10	}
11}

playground


後は、mapに対してよしなに処理してやればOKです。

やはり、ちゃんと構造体を作って、
フィールドで型を指定してやるのが理想だと思います。

しかし、正常時と異常時でレスポンス構造が大きく変わる場合などは、
一旦、map型に変換して、正常時用か異常時用かを判断するのもありかなと思います。

正常時と異常時の両方に対応できるでっかい構造体を作ってもいいですが、
無駄が多くなりがちですしね。

その状況に合わせて、使い分けれると良さそうですね👍
みなさんどうしているのか聞いてみたいですね〜
